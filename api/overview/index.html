    Overview

    The Azuma HTTP API is used to interact with parts of the Azuma server.

    HTTP API Routes#

    HTTP API Route documentation consists of the request description, the response description and possible errors you may encounter. Request and response descriptions contain JSON bodies when applicable, those define available keys and values in pseudo-types, followed by an example. Some errors can happen on every route, those are described in Errors.

    Errors#

    When an error occurs, the server will respond with an error status code and a JSON body including a message, in case the error is not clear, it is shown in the following codeblock.

    {
    "message": "String" // "ALREADY_EXISTS"
    }

    400 Bad Request#

    The request you're sending is invalid. Please double-check that you are sending it as described in it's documentation. If you've done that and it still doesn't work, create an issue in the backend repository.

    500 Internal Server Error#

    This is most likely not your fault. Contact the Azuma instance hoster.

    Sessions#

    All routes except for HTTP API Index, Register User and Login User require authentication, obtained through the latter two routes. This token is used as a bearer-token in the Authorization header, like shown in the following codeblock.

    Authorization: Bearer 318cd399-f47f-478f-8d09-c9c414c3750d

    HTTP API Index#

    This route returns information about the Azuma instance.

    GET /
